Acumed Digital Surgery — Design System

A reference for every UI component and pattern in the ADS prototypes. Use this page to inspect the visual specifications, copy markup into new screens, and verify that work stays consistent with the brand tokens.

Colors

All colors live in styles/tokens.css. Use brand tokens for everything; the few raw hex values (status pills, On Hold gold, Rework orange) are intentional exceptions documented in the token file.

Lane status filter

Multi-select filter. Attached to every swim-lane head and to every list-view section head. Clicking a status chip toggles it on/off; the lane shows the union of every active chip. Clicking "All" clears the set and restores every card. With no chips active, "All" is implicitly active. Empty statuses are dimmed but still selectable. Menu stays open while selecting — outside click or Esc closes.

Open — nothing selected (= All)
Open — multiple chips active

QC Pass / Fail

Two-button pattern on every Quality Check step. Default: Fail (secondary, left) + Pass (primary, right). Clicking Fail reveals a "Return case to" + "Reason" sub-form and swaps the actions to Back + Submit (danger). Back returns to the default view; Submit commits the rejection.

Default state
Fail mode (rejection being composed)

Confirm modal

Branded confirmation dialog. Replaces native window.confirm for destructive actions (file delete) and important transitions (move-to-next, surgeon reject). Call showConfirm({title, body, confirmLabel, confirmVariant, onConfirm}) — variant defaults to danger (red Delete-style button), pass primary for non-destructive advances.

Surgeon Approval — 3-outcome decision

The active Surgeon Approval pp-step has three possible outcomes: Approve (advances In Process), Internal reject (returns to Request approval — Rework), Surgeon reject (case is Blocked). Default view shows Approve + Reject. Reject swaps the panel for a radio choice (internal vs surgeon) + reason textarea; Submit fires a confirm dialog with copy that mirrors the chosen outcome.
